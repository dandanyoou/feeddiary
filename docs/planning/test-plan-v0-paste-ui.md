## Affected Pages/Routes

- **`/` (root)** — paste form, validation, submit, loading states, error display
- **`/c/[slug]`** — share page, card PNG render, OG meta, download + copy buttons
- **`/c/[invalid-slug]`** — 404 page
- **`/api/ingest` (existing, called from /)** — verify 200/400/429/500 paths surface correctly in UI
- **`/api/og/[slug]` (existing, embedded in /c/[slug])** — verify image loads with correct dimensions

## Key Interactions to Verify

- Paste 5-50 valid YouTube URLs → submit enabled → loading messages cycle → redirect to `/c/[slug]`
- Paste fewer than 5 URLs → submit disabled → helper text shows "5개 이상 필요"
- Paste more than 50 URLs → submit disabled → helper text shows "50개 이하 필요"
- Paste non-YouTube text → 0 detected → submit disabled
- Submit while rate-limited → see "1시간에 5번까지만…약 N분 후" message
- Submit with network offline → see "연결이 불안정해요" message
- On `/c/[slug]` click "이미지 저장" → PNG downloads
- On `/c/[slug]` click "링크 복사" → clipboard contains share URL → confirmation toast
- On `/c/zzzzzzzz` → 404 page with "카드 만들러 가기" link back to `/`
- Share `/c/[slug]` URL on Twitter/Discord → preview card unfurls (og:image, twitter:card)

## Edge Cases

- Submit during loading state → button disabled, no duplicate request
- User navigates away mid-submit → no console errors, timers cleaned up
- Clipboard API permission denied (Safari quirk) → graceful fallback message
- Double-click submit → debounced by disabled state
- Empty textarea → submit disabled, no validation error shown until user types
- Paste with mixed valid + invalid lines (e.g., 5 YouTube + 3 random URLs) → 5 detected, submit enabled
- Paste duplicate URLs → dedupe shows lower count
- `/c/[slug]` opened directly without going through `/` → renders correctly (independent of session)
- Server returns 200 but malformed body → catch + generic error
- PNG fails to load on share page → `<img>` shows alt text "이번 주 카드"

## Critical Paths

- **Golden path:** `/` → paste 10 URLs → submit → loading (3 phases) → redirect to `/c/[slug]` → PNG visible → click "링크 복사" → clipboard has URL
- **404 path:** `/c/zzzzzzzz` → renders 404 page → click "카드 만들러 가기" → back to `/`
- **Rate limit recovery:** trigger 5 submits → 6th shows KR error with retryAfter → wait → succeeds (manual test only, e2e too slow)
- **Validation path:** `/` → paste 3 URLs → submit disabled + helper text → add 2 more → submit enabled
