the card. After this lands, design doc Gate 1 ("working v0 web app deployed, single
card mode, YouTube paste-link ingestion only") is complete and Gate 2 (launch +

- **Korean PIPA privacy policy page** — not needed until Korean-language launch (design
  doc Gate 2+).
- **Public-share OG meta for Instagram Stories** — IG Stories does not unfurl URLs;
  the share button copies the link, user uploads PNG separately.

## Architecture decisions (this review)

### D1 — Submit flow: `POST /api/ingest` → 200 `{slug}` → `router.push('/c/${slug}')`

After successful submit, client redirects to the share page. Single entry surface,
single share-ready URL, native back-button works.

```
USER FLOW
   /
   ├── paste textarea (n URLs detected hint)
   ├── [submit] ──► POST /api/ingest
   │                      ├── 200 {slug}    ──► router.push('/c/${slug}')   ──► /c/abc12345
   │                      ├── 429           ──► inline error w/ retryAfter (Korean)
   │                      ├── 400           ──► inline error w/ KR message
   │                      └── 500/network   ──► inline generic error (Korean)
   └── timeline messages while pending      (0-2s / 2-7s / 7s+)
```

### D2 — Share page renders PNG-only

`<img src="/api/og/[slug]">` + two buttons: "이미지 저장" (download PNG) and "링크 복사"
(clipboard API). No video list, no "remake" CTA, no embedded UI chrome that competes
with the card. Aligns with design doc Approach C: "the deliverable is one tweetable
screenshot."

### D3 — Loading UX: time-based timeline messages

Three phases switched by `setTimeout`:

```
t=0s    "영상 제목 가져오는 중..."          (oEmbed phase)
t=2s    "AI가 카테고리 분석 중..."           (Claude phase)
t=7s    "카드 그리는 중..."                  (slug + insert phase)
```

Not real server-sent events — fake timeline that approximates the actual ingest
pipeline. Cancel timers on success/error. SSE-based real progress is a v1 follow-up.

### D4 — Server error codes → Korean messages in `lib/error-messages.ts`

One canonical mapping table, importable from any future component. Pure function,
fully unit-testable. Avoids duplicating the switch inside `PasteForm`.

```typescript
// lib/error-messages.ts
export function errorCodeToMessage(
  code: string,
  extra?: { retryAfterMs?: number }
): string { ... }
```

### D5 — Test stack: Playwright (3 critical flows) + Vitest+RTL (units)

Playwright covers paste → loading → card golden path, invalid slug (404), and
client-side validation. Vitest+RTL covers `PasteForm` states, `errorCodeToMessage`
table, share-page server-component data flow.

## Implementation details (per file)

### `lib/error-messages.ts`

```typescript
const HOUR_MS = 60 * 60 * 1000;

export interface ErrorExtra {
  retryAfterMs?: number;
  message?: string;
}

export function errorCodeToMessage(code: string, extra: ErrorExtra = {}): string {
  switch (code) {
    case 'RATE_LIMITED': {
      const mins = Math.ceil((extra.retryAfterMs ?? HOUR_MS) / 60000);
      return `1시간에 5번까지만 만들 수 있어요. 약 ${mins}분 후에 다시 시도해주세요.`;
    }
    case 'AT_LEAST_5_VALID_YOUTUBE_URLS':
      return 'YouTube 링크가 5개 이상 필요해요. 더 추가해주세요.';
    case 'MAX_50_URLS':
      return '한 번에 최대 50개까지만 가능해요. 줄여서 다시 시도해주세요.';
    case 'INVALID_JSON':
    case 'URLS_MUST_BE_ARRAY':
    case 'URLS_MUST_BE_STRINGS':
      return '입력이 올바르지 않아요. 페이지를 새로고침 후 다시 시도해주세요.';
    case 'METRICS_FAILED':
    case 'DB_WRITE_FAILED':
      return '카드를 만들지 못했어요. 잠시 후 다시 시도해주세요.';
    case 'NETWORK_ERROR':
      return '연결이 불안정해요. 네트워크를 확인해주세요.';
    default:
      return '문제가 발생했어요. 잠시 후 다시 시도해주세요.';
  }
}
```

### `app/components/paste-form.tsx` ('use client')

Pseudocode (full implementation):

```typescript
'use client';
import { useState, useRef, useEffect } from 'react';
import { useRouter } from 'next/navigation';
import { parseUrls, dedupeVideos } from '@/lib/youtube';
import { errorCodeToMessage } from '@/lib/error-messages';

const MIN_URLS = 5;
const MAX_URLS = 50;

const TIMELINE = [
  { at: 0,    msg: '영상 제목 가져오는 중...' },
  { at: 2000, msg: 'AI가 카테고리 분석 중...' },
  { at: 7000, msg: '카드 그리는 중...' },
];

export function PasteForm() {
  const router = useRouter();
  const [text, setText] = useState('');
  const [submitting, setSubmitting] = useState(false);
  const [error, setError] = useState<string | null>(null);
  const [timelineIdx, setTimelineIdx] = useState(0);
  const timers = useRef<ReturnType<typeof setTimeout>[]>([]);

  const detected = dedupeVideos(parseUrls(text)).length;
  const canSubmit = !submitting && detected >= MIN_URLS && detected <= MAX_URLS;

  function startTimeline() {
    setTimelineIdx(0);
    TIMELINE.slice(1).forEach((step, i) => {
      timers.current.push(setTimeout(() => setTimelineIdx(i + 1), step.at));
    });
  }
  function clearTimeline() {
    timers.current.forEach(clearTimeout);
    timers.current = [];
  }
  useEffect(() => clearTimeline, []); // cleanup on unmount

  async function handleSubmit(e: React.FormEvent) {
    e.preventDefault();
    setError(null);
    setSubmitting(true);
    startTimeline();
    try {
      const res = await fetch('/api/ingest', {
        method: 'POST',
        headers: { 'content-type': 'application/json' },
        body: JSON.stringify({ urls: text.split(/\s+/).filter(Boolean) }),
      });
      if (!res.ok) {
        const body = await res.json().catch(() => ({}));
        setError(errorCodeToMessage(body.error ?? 'UNKNOWN', body));
        return;
      }
      const { slug } = await res.json();
      router.push(`/c/${slug}`);
    } catch {
      setError(errorCodeToMessage('NETWORK_ERROR'));
    } finally {
      clearTimeline();
      setSubmitting(false);
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <textarea
        value={text}
        onChange={(e) => setText(e.target.value)}
        placeholder="YouTube 링크를 한 줄에 하나씩 붙여넣어주세요 (5-50개)"
        disabled={submitting}
        aria-label="YouTube URLs"
      />
      <div aria-live="polite">
        {!submitting && (detected > 0
          ? `${detected}개 감지됨 (5-50개 필요)`
          : '아직 감지된 링크가 없어요')}
        {submitting && TIMELINE[timelineIdx].msg}
      </div>
      <button type="submit" disabled={!canSubmit}>
        {submitting ? '만드는 중...' : '카드 만들기'}
      </button>
      {error && <p role="alert">{error}</p>}
    </form>
  );
}
```

## Failure modes

| New codepath                   | Realistic failure                   | Test? | Error handling? | User sees? |
| ------------------------------ | ----------------------------------- | ----- | --------------- | ---------- |
| `POST /api/ingest` 429 path    | Vercel cold-start resets in-memory  | yes   | yes (D4 msg)    | clear KR error |
| `POST /api/ingest` 500         | Claude API outage                   | yes   | yes (D4 msg)    | clear KR error |
| `fetch` network throw          | User offline mid-submit             | yes   | yes (NETWORK_ERROR) | clear KR error |
| `getCardWithItems` DB throws   | Supabase outage                     | yes   | Next.js error boundary | generic 500 |
| Invalid slug `/c/xxxx`         | User mistypes URL or expired card   | yes   | notFound()      | 404 page |
| `router.push` race after unmount | User navigates away mid-submit    | yes   | useEffect cleanup clears timers | no-op |
| Clipboard API permission denied | Safari permission UX             | no    | catch + toast   | "복사 실패, 직접 선택해주세요" |
| Download blob CORS issue       | `/api/og/[slug]` same origin so OK  | no    | n/a             | n/a |

**No critical gaps** — every codepath has test + error handling + user-visible error.

## Worktree parallelization strategy

Two lanes, mostly independent:

| Step | Modules touched                             | Depends on |
| ---- | ------------------------------------------- | ---------- |
| L1.A | `lib/` (error-messages + test)              | —          |
| L1.B | `app/components/` (PasteForm + test)        | L1.A (imports error-messages) |
| L1.C | `app/page.tsx`                              | L1.B       |
| L2.A | `app/c/[slug]/` (page + not-found + tests)  | —          |
| L3   | `e2e/` Playwright (setup + 3 specs) + config | L1.C + L2.A |

**Execution order:** Launch L1.A and L2.A in parallel worktrees. L1.A completes → L1.B
sequentially. L1.B → L1.C sequentially. Merge L1 and L2 lanes. Then L3.

No module-directory conflicts between L1 and L2. CC time estimate: L1+L2 in parallel
~20 min, L3 ~15 min. Total ~35 min CC time / ~6h human.
